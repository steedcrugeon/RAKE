General Science Notes for the GoingUndground Kerbal Space Program Mod

These notes are for support of the development collaboration and references purposes.
Notes are written based aorund KSP 1.6.1 (version used at time of development).
No additional '.dll' files are required to use the science associated with this mod
The design aim for GoingUndground is to be compatible with the 'Stock' game.

Background: GoingUndground is a KSP mod which adds 'subterranean habitats' to the game
which includes parts, and science! These are intended for mid - late game use.

Three type of Science Definition File 'suffix'; Basic , Detailed and Exhaustive.
It should be noted that for the purposes of this mod we are only intersted in 'SrfLanded'
there will be no science conducted in space or atmospheric flight.

--- Science PartModule Config Example ---

MODULE
	{
		name = ModuleScienceExperiment	// Stock KSP ModuleScienceExperiment
	
		experimentID = exampleScienceExperiment // unique ID calls to the corresponding science in the "ScienceDef.cfg" file
	
		experimentActionName = #autoLOC_MOD_000_experimentActionName // "Perfrom Experiment"
		resetActionName = #autoLOC_MOD_001_resetActionName // "Reset Experiment"
		
		useStaging = False
		useActionGroups = True
		hideUIwhenUnavailable = False	
		
		xmitDataScalar = 0.45 // percentage of science value retained when 'transmitting' the data back to Kerbin
		dataIsCollectable = True // Whether the Science data can be 'removed' from the experiment for storage elsewhere (capsule).
		collectActionName = #autoLOC_MOD_002_collectActionName // "Collect Experiment Data"
		interactionRange = 1.2 // How close the Kerbal muse be to interact with the Experiment
		usageReqMaskInternal = 1 // What type of Kerbal can perform the experiment from within the craft it's attached to.
		usageReqMaskExternal = 8 // What type of Kerbal can perform the expermient extenal to the experiment (outside the craft).
		rerunnable = True // Can the expermient be done again without a scientist restting it.
	}

// Basic GU Crew Report (basicGUcrewReport)

{
	name = ModuleScienceExperiment	// Stock KSP ModuleScienceExperiment

	experimentID = basicGUcrewReport // unique ID calls to the corresponding science in the "ScienceDef.cfg" file

	experimentActionName = #autoLOC_RAKE_0701 // Carry out basic sub-surface crew observation
	resetActionName = #autoLOC_RAKE_0702 // Expunge observation records
	reveiwAction Name = #autoLOC_RAKE_0703 // Review observation report
	
	useStaging = False
	useActionGroups = False
	hideUIwhenUnavailable = True	
	
	xmitDataScalar = 1 // 100% of science value retained when 'transmitting' the data back to Kerbin
	dataIsCollectable = True // Science data can be 'removed' from the experiment for storage elsewhere.
	collectActionName = #autoLOC_RAKE_0704 // Collect observation report
	situationMask = 1 // This experiment can only be run when vessel is  landed on the surface
	usageReqMaskInternal = 4 // Crew must be in this part to run the experiment
	usageReqMaskExternal = -1 // This experiment cannot be run from outside.
	rerunnable = True // Can the expermient be done again without a scientist reseting it.
}
